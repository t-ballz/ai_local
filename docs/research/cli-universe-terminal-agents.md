# CLI-Universe: Verifiable Synthetic Data for Terminal Agents

> Source: [arXiv:2606.22883](https://arxiv.org/abs/2606.22883) · Jun 2026  
> Authors: Zhanbo Hua, Yifan Yao, Weihao Xie, Yongchi Zhao, Minghao Liu, Ruizhi Qiu, Zhewei Huang, Zun Wang, Yiyan Ji, Yunhai Ye, Letian Zhu, Xinping Lei, Han Li, Zhiyuan Ma, Zili Wang, Zhaoxiang Zhang, Jiaheng Liu

## TL;DR

CLI-Universe is a pipeline for synthesizing high-quality terminal agent training data using a 4-dimensional CLI capability taxonomy, evidence-guided task grounding, and multi-stage executable verification in Docker containers. Fine-tuning Qwen3-32B on just 6,000 trajectories (CLI-Universe-6K) achieves state-of-the-art for open-source models ≤32B on Terminal-Bench 2.0 with 33.4%.

---

## The problem

LLM-based terminal agents are useful for DevOps, data engineering, and scientific computing — but training data is scarce. Real CLI task datasets are small, narrow (bash scripting only), and hard to verify programmatically. Synthetic data generation without verification produces trajectories with wrong answers, which corrupt training.

---

## How it works

**4-dimensional CLI taxonomy**: Every task is positioned along:
1. **Domain** (systems administration, data processing, development, scientific computing, etc.)
2. **Skill type** (file manipulation, process control, networking, package management, etc.)
3. **Capability** (single-step, multi-step, conditional, error recovery, etc.)
4. **Engineering pillar** (correctness, efficiency, security, maintainability)

Tasks are generated by sampling combinations across these dimensions, ensuring diversity across the full capability space rather than concentrating on common bash patterns.

**Evidence-guided research**: For each task blueprint, the system researches relevant documentation, man pages, and reference implementations to ground the task in real technical environments.

**Multi-stage verification** (the key innovation): Candidate tasks run through:
1. Rubric-gated test construction — auto-generates test cases from task specification
2. Hint-conditional filtering — verifies the task is solvable with the given hints
3. Strict fail-to-pass checking — confirms the baseline (no solution) fails and the reference solution passes

~2/3 of generated candidates are rejected. The result: 6,000 high-confidence, verified trajectories.

## Results

| Model | Terminal-Bench 2.0 | Notes |
|---|---|---|
| Qwen3-32B + CLI-Universe-6K | **33.4%** | SOTA for open-source ≤32B |
| Previous best (open ≤32B) | < 33.4% | |

6,000 trajectories — exceptional data efficiency for SOTA results at this scale.

---

## Why it matters for local AI

- **Terminal agents are practically useful**: DevOps automation, data pipeline construction, sysadmin tasks — all benefit from a capable CLI agent running locally
- **Verified synthetic data pattern**: The rubric-gated + fail-to-pass verification approach is generalizable to other code/tool-use domains (SQL, Python scripting, API usage)
- **Qwen3-32B base**: The fine-tuned model is built on an open-weight model — the approach is reproducible with your own domain-specific taxonomy
- **6K is tractable**: Generating 6,000 verified trajectories is feasible for research teams without massive compute

---

## Limitations

- Terminal-Bench 2.0 is the primary benchmark — real-world task distribution may differ
- Verification pipeline requires Docker — adds infrastructure overhead during data generation
- Domain coverage biased toward Unix/Linux CLI; Windows PowerShell or cross-platform tools are underrepresented
